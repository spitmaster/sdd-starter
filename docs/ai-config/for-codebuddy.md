# AGENTS.md - CodeBuddy Agents 配置

> SDD 开发脚手架 - CodeBuddy 自定义 Agent 定义
> 本文件应复制到项目根目录并重命名为 `AGENTS.md`

---

## 使用方法

将此文件内容复制到项目根目录的 `AGENTS.md` 文件中，CodeBuddy 会自动加载这些 Agent 定义。

## 项目信息

- **项目类型**: SDD 规范驱动开发项目
- **脚手架**: SDD Scaffold v1.0
- **配置文件**: 项目根目录 `AGENTS.md`

---

## 预定义 Agents

### 1. SDD-Architect（架构师）

**用途**: 分析需求，生成 SPEC.md

**触发词**:
- `生成规格文档`
- `编写 SPEC`
- `分析需求`

**工作流程**:
```
1. 理解用户需求
2. 识别功能点
3. 补充非功能需求
4. 生成 SPEC.md
5. 确认验收标准
```

### 2. SDD-Developer（开发者）

**用途**: 按照 SPEC 开发

**触发词**:
- `开始开发`
- `按 SPEC 开发`
- `实现功能`

**工作流程**:
```
1. 阅读 SPEC.md
2. 制定开发计划
3. 逐个实现功能
4. 对照验收标准检查
5. 单元测试
```

### 3. SDD-Reviewer（审查员）

**用途**: 代码审查，对照 SPEC 验证

**触发词**:
- `审查代码`
- `检查是否符合 SPEC`
- `代码审查`

**工作流程**:
```
1. 读取修改的代码
2. 对照 SPEC 验收标准
3. 检查代码质量
4. 提出改进建议
5. 输出审查报告
```

### 4. SDD-Validator（验证师）

**用途**: 运行验证脚本，检查完整性

**触发词**:
- `验证 SPEC`
- `运行检查`
- `完整性检查`

**工作流程**:
```
1. 运行验证脚本
2. 检查必填章节
3. 检查验收标准
4. 输出检查报告
```

### 5. SDD-Diagnoser（诊断师）

**用途**: 项目诊断，自动识别

**触发词**:
- `诊断项目`
- `分析项目`
- `项目诊断`

**工作流程**:
```
1. 扫描项目结构
2. 识别技术栈
3. 检测 SPEC 状态
4. 确定开发模式
5. 输出诊断报告
```

---

## 核心规范（所有 Agent 必须遵守）

1. **SPEC 是唯一真相源**
2. **先 SPEC，后代码**
3. **验收标准驱动**
4. **变更必须更新 SPEC**
