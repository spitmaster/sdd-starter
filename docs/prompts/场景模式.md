# 场景模式提示词

> 提供给 AI 的标准化提示词

---

## 场景 A：PRD 导入

```
请作为 SDD 架构师，分析以下需求并生成 SPEC.md：

## 你的任务
1. 分析用户提供的内容，提取关键信息
2. 识别遗漏或不明确的地方，询问用户确认
3. 补充非功能需求（性能、安全、兼容性等）
4. 生成完整的 SPEC.md

## PRD 内容
[粘贴用户的 PRD 或需求描述]

## 输出要求
- 包含所有必要的章节
- 每个功能有明确的验收标准
- 验收标准必须可测试
- 如有疑问，先询问再继续
```

---

## 场景 B：敏捷探索

```
请以敏捷探索模式启动这个项目：

## 你的任务
1. 做一个最小可用的原型（MVP）
   - 只包含核心功能
   - 不需要完美，能演示即可
   - 完成后主动让用户体验

2. 收集反馈并迭代
   - 询问用户体验感受
   - 根据反馈调整/新增功能
   - 保持快速迭代

3. 整理文档
   - 项目完成后，整理 SPEC.md
   - 包含已完成的功能
   - 列出可能的扩展点

## 用户想法
[描述用户的想法]

## 输出要求
- 快速交付第一个可用的版本
- 每次迭代都让用户确认
- 最终交付包含 SPEC.md
```

---

## 场景 C：项目接续

```
请以项目诊断模式启动：

## 你的任务
1. 自动扫描项目
   - 识别技术栈（语言、框架、依赖）
   - 识别项目架构
   - 检测是否有 SPEC.md

2. 分析代码
   - 识别代码风格
   - 检测潜在问题
   - 评估代码复杂度

3. 确定开发模式
   - 如果有 SPEC → 按 SPEC 继续
   - 如果没有 SPEC → 生成后继续
   - 如果有 bug → 分析并修复
   - 如果要加功能 → 分析架构后实现

4. 输出诊断报告
   - 项目概况
   - 发现的问题
   - 接下来的开发建议

## 用户需求（可选）
[描述用户的具体需求，如：修 bug、加功能、重构]
```
