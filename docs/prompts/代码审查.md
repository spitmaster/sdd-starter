# 代码审查提示词

```
请对照 SPEC.md 审查代码：

## 审查范围
- 新增/修改的代码文件
- 相关测试文件
- 相关配置文件

## 审查维度

### 1. 功能完整性
- [ ] 是否实现了所有需求？
- [ ] 是否有遗漏的功能点？
- [ ] 边界情况是否处理？

### 2. 代码质量
- [ ] 代码是否清晰易读？
- [ ] 命名是否规范？
- [ ] 是否有重复代码？

### 3. 安全性
- [ ] 用户输入是否验证？
- [ ] 是否有安全漏洞？

### 4. 性能
- [ ] 是否有明显性能问题？
- [ ] 数据库查询是否优化？
