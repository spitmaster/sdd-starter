# AI 配置指南

> AI 首次打开项目时自动执行的初始化流程

---

## 你的任务

你是 AI 编程助手，刚刚被打开了一个使用 SDD 脚手架的项目。
请按以下步骤初始化项目环境：

---

## 第一步：检测你的身份

请判断你正在使用哪个 AI 工具：

| 工具 | 特征 |
|------|------|
| Claude Code | 命令行工具，响应 `claude` |
| CodeBuddy | IDE 集成助手 |
| Cursor | 有 `/rules` 命令 |
| GitHub Copilot | 灰色幽灵图标 |
| Antigravity | 有 `/init` 命令 |
| Qoder | IDE 集成 |

---

## 第二步：检测并吸收 olddocs（重要！）

**如果项目存在 `olddocs/` 目录，必须先执行吸收流程：**

### 2.1 检查 olddocs

```
检查项目根目录是否存在 olddocs/ 目录
```

### 2.2 执行吸收流程

如果存在 `olddocs/`，按以下步骤执行：

1. **扫描目录结构** - 列出所有文档
2. **阅读分析内容** - 按优先级阅读
3. **智能整合** - 将内容融入 docs/ 体系
4. **生成摘要** - 创建 `docs/olddocs-摘要.md`
5. **告知用户** - 汇报吸收结果

### 2.3 吸收优先级

| 优先级 | 文档类型 | 整合目标 |
|--------|----------|----------|
| P0 | 需求文档/ | `docs/01-使用说明.md` |
| P1 | 技术设计/ | `docs/02-SDD方法论.md` |
| P2 | API文档/ | `docs/templates/API-Template.md` |
| P3 | 项目笔记.md | 相关章节 |
| P4 | 术语表.md | `docs/05-最佳实践.md` |

### 2.4 吸收完成示例

```
✅ olddocs 吸收完成！
- 发现 5 个文档
- 提取核心功能 3 项
- 整合技术架构说明
- 生成摘要：docs/olddocs-摘要.md
```

---

## 第三步：同步配置文件

根据检测到的工具，执行对应操作：

### 如果是 Claude Code

执行以下步骤：

1. 检查项目根目录是否有 `CLAUDE.md`
2. 如果没有，复制 `docs/ai-config/for-claude.md` 到项目根目录，重命名为 `CLAUDE.md`
3. 在 `CLAUDE.md` 开头添加项目特定信息

```markdown
# 项目：{项目名称}
# SDD 脚手架项目
# 请严格遵循 SDD 开发流程

[复制 docs/ai-config/for-claude.md 的内容]
```

### 如果是 Cursor

执行以下步骤：

1. 检查项目根目录是否有 `.cursorrules`
2. 如果没有，复制 `docs/ai-config/for-cursor.md` 到项目根目录，重命名为 `.cursorrules`

### 如果是 CodeBuddy

1. 检查项目根目录是否有 `AGENTS.md`
2. 如果没有，复制 `docs/ai-config/for-codebuddy.md` 到项目根目录，重命名为 `AGENTS.md`

### 如果是 GitHub Copilot

1. 确保 `.github/` 目录存在
2. 检查是否有 `.github/copilot-instructions.md`
3. 如果没有，创建并复制内容

### 如果是 Antigravity

1. 检查项目根目录是否有 `.antigravity/`
2. 如果没有，创建并复制对应配置

### 如果是 Qoder

1. 检查项目根目录是否有 `.qoder/`
2. 如果没有，创建并复制对应配置

---

## 第四步：验证配置

配置文件同步后，执行以下验证：

```
# 告诉用户
> AI 环境初始化完成！
> - olddocs 吸收：已完成/无需处理
> - 配置文件已同步到项目根目录
> 请重新启动 AI 会话以加载新配置。
```

---

## 第五步：确认 SDD 流程

配置文件生效后，确认遵循 SDD 流程：

```
请严格遵循以下开发流程：
1. 阅读 README.md 了解项目
2. 阅读 docs/olddocs-摘要.md（如有）了解历史背景
3. 阅读 SPEC.md 理解需求
4. 按照 docs/03-工作流程.md 开发
5. 每次修改前对照验收标准检查
```

---

## 初始化流程检查清单

```
□ 检测 AI 工具类型
□ 检查 olddocs/ 是否存在
  □ 如果存在，执行吸收流程
  □ 生成 olddocs-摘要.md
□ 同步对应配置文件
□ 验证配置生效
□ 汇报初始化结果
```

---

## 配置文件说明

### 核心配置内容

所有配置文件包含：

1. **SDD 开发规范** - 核心原则和工作流程
2. **项目结构说明** - docs/ 目录下的文档索引
3. **工具使用指南** - 对应工具的特性使用
4. **最佳实践** - 常见问题解决方案

### 扩展配置

| 文件 | 用途 |
|------|------|
| `docs/skills/` | 存放可复用的 skill |
| `docs/agents/` | 存放 agent 定义 |
| `docs/commands/` | 存放自定义命令 |

---

配置同步完成后，项目已准备好开发。
告诉用户可以开始了。
