# AI 工具使用指南

> 在各种 AI 编程工具中使用 SDD 流程

---

## 通用加载命令

在任何 AI 工具中，首先加载 SDD 流程：

```
请严格遵循 SDD 开发流程：
1. 先阅读 docs/01-使用说明.md 了解核心规范
2. 阅读 SPEC.md 理解项目需求
3. 按照 docs/03-工作流程.md 执行开发
4. 每次修改前对照 SPEC 中的验收标准检查
```

## Cursor

### 加载方式
在 `.cursorrules` 文件中添加：

```markdown
# SDD 开发规范
- 严格遵循 docs/03-工作流程.md
- 每次修改前对照 SPEC.md 验收标准
- 不添加 SPEC 外的功能
```

或在对话中输入：
```
> 请阅读 .cursorrules 中的规范，并严格执行
```

### 特点
- Tab 自动补全
- 支持多文件编辑

## 场景化使用

### 场景 A：有 PRD

```bash
> 请分析 docs/prd.md 并生成 SPEC.md
> 完成后告诉我遗漏了哪些细节
```

### 场景 B：模糊想法

```bash
> 我想做个记账应用，但具体还不知道
> 请先做个最小原型让我看看
```

### 场景 C：已有项目

```bash
> 请诊断当前项目
> 阅读 docs/08-项目诊断器.md 的规则
```

## AI 提示词模板

### 启动项目
```
请以 SDD 模式启动项目：
1. 阅读 docs/01-使用说明.md
2. 诊断项目情况
3. 制定开发计划
```

### 开发过程中
```
请严格按照 SPEC.md 开发：
- [ ] 功能见第 2 节
- [ ] UI 规范见第 4 节
- [ ] 验收标准见第 7 节
```

### 代码审查
```
请对照 SPEC.md 审查代码：
1. 检查是否满足所有验收标准
2. 检查是否有遗漏的功能点
3. 提出改进建议
```

## 常见问题

**Q: AI 不按 SDD 流程走怎么办？**
> 重新发送加载命令，明确要求遵循流程。

**Q: AI 生成的内容不符合 SPEC 怎么办？**
> 指出具体不符合的地方，让它修正。

**Q: 如何让 AI 主动检查？**
> 添加指令："每完成一个功能，主动对照验收标准检查"
