# 小程序开发指南

> 微信小程序专项开发规范

---

## 小程序项目结构

```
project/
├── app.js                    # 应用入口
├── app.json                  # 应用配置
├── app.wxss                  # 全局样式
├── pages/                    # 页面目录
│   ├── index/
│   ├── detail/
│   └── mine/
├── components/              # 组件目录
├── utils/                   # 工具函数
├── services/                # API 服务
└── assets/                  # 静态资源
```

---

## SPEC 模板（小程序版）

```markdown
# [项目名称] 小程序规格说明书

## 1. 项目概述
- **项目名称**:
- **小程序名称**:
- **AppID**:
- **核心功能**:

## 2. 页面清单

| 页面 | 路径 | 功能描述 |
|------|------|----------|
| 首页 | pages/index/index | 展示核心内容 |
| 详情页 | pages/detail/detail | 详情展示 |
| 我的 | pages/mine/mine | 用户中心 |

## 3. 功能需求

### 3.1 首页功能
- Banner 轮播（高度、间隔、点击跳转）
- 分类导航（图标数量、布局）
- 商品列表（下拉刷新、上拉加载更多）

## 4. 页面路由

```json
{
  "pages": [
    "pages/index/index",
    "pages/detail/detail",
    "pages/mine/mine"
  ],
  "tabBar": {
    "list": [
      { "pagePath": "pages/index/index", "text": "首页" },
      { "pagePath": "pages/mine/mine", "text": "我的" }
    ]
  }
}
```

## 5. 组件规范

```javascript
Component({
  properties: {
    title: { type: String, value: '' }
  },
  data: {},
  methods: {}
})
```

## 6. API 设计

| 接口 | 方法 | 路径 | 说明 |
|------|------|------|------|
| 获取首页数据 | GET | /api/home/index | Banner、分类 |
| 获取商品详情 | GET | /api/goods/detail | 商品 ID |

## 7. UI 规范

### 尺寸单位
- 使用 **rpx** 作为单位
- 设计稿宽度: 750px

### 颜色规范

```css
--primary-color: #ff6b6b;
--success-color: #52c41a;
--text-color: #333333;
--bg-color: #f5f5f5;
```

## 8. 验收标准

| 功能 | 验收标准 | 测试方法 |
|------|----------|----------|
| 首页加载 | 进入首页 3 秒内展示内容 | 掐表计时 |
| 下拉刷新 | 下拉显示 loading，松开刷新数据 | 手动测试 |
| 页面跳转 | 点击跳转正常，返回正确 | 手动测试 |

## 项目诊断规则（小程序版）

### 自动识别

| 检测项 | 小程序特征 |
|--------|-----------|
| `app.json` | 微信小程序 |
| `project.config.json` | 微信小程序 |
| `swan.json` | 百度小程序 |
| `tt.json` | 字节小程序 |
| `uni.scss` | uni-app |
| `manifest.json` | uni-app 原生 |

### 小程序诊断输出

```markdown
## 项目诊断报告

### 项目类型
- **类型**: 微信小程序
- **框架**: 原生开发 / Taro / uni-app

### 技术栈
- **UI 框架**: WeUI / Vant / 自定义
- **状态管理**: MobX / Redux / Hooks

### 建议
1. 补充 pages.json 路由配置
2. 统一组件开发规范
3. 建立请求封装
```
