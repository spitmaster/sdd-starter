# olddocs - 历史文档目录

> 本目录用于存放项目迁移前的历史文档。AI 启动时会自动扫描并吸收其中的信息。

---

## 📋 目录说明

| 文件/目录 | 用途 | AI 吸收后会整合到 |
|-----------|------|------------------|
| `需求文档/` | 原始需求、PRD、用户故事 | `docs/01-使用说明.md` |
| `技术设计/` | 架构设计、API设计、数据库设计 | `docs/02-SDD方法论.md` |
| `API文档/` | 接口文档、SDK文档 | `docs/templates/API-Template.md` |
| `项目笔记.md` | 零散笔记、会议纪要、决策记录 | 融入相关章节 |
| `术语表.md` | 业务术语、技术术语解释 | `docs/05-最佳实践.md` |
| `其他/` | 无法归类的文档 | 单独章节 |

---

## ⚡ 使用方法

### 1. 迁移老文档

将项目历史文档按类型放入对应目录：

```
olddocs/
├── 需求文档/
│   ├── PRD-2024.md
│   └── 用户故事.md
├── 技术设计/
│   ├── 架构设计.md
│   └── 数据库设计.md
├── API文档/
│   └── 接口文档.md
├── 项目笔记.md
├── 术语表.md
└── 其他/
    └── 一些参考资料.md
```

### 2. AI 自动吸收

AI 工具启动后，会自动：
1. 扫描 `olddocs/` 目录
2. 阅读所有文档内容
3. 将有用信息整合到 `docs/` 体系
4. 在 `docs/olddocs-摘要.md` 生成吸收摘要

---

## 🔄 吸收规则

| 原始文档类型 | 吸收方式 |
|-------------|---------|
| 需求文档 | 提取核心需求，融入 SDD 规格说明 |
| 技术设计 | 提取架构要点，融入方法论文档 |
| API文档 | 提取接口规范，创建 API 章节 |
| 项目笔记 | 提取决策记录和上下文 |
| 术语表 | 创建术语章节或词汇表 |

---

## 📌 注意事项

- **不要删除原始文档** - 保留在 olddocs/ 便于追溯
- **AI 会智能合并** - 不会简单覆盖，会整合到相关章节
- **需要人工确认** - 吸收完成后会生成摘要供你确认
